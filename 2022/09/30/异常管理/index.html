异常管理

Created2022-09-30|Updated2022-10-17
|Word count:805|Reading time:3min|Post View:

异常:java.net.SocketTimeoutException: Read timed out Socket 读写超时

原因
微服务间调用其实走的是http请求，debug了一下默认的ReadTimeout时间为5s，ConnectTimeout时间为2s

解决方法

  1. 换个好点网络环境。(实测有效),原则是”越快拿到服务就越好”
  2. 延长 微服务 之间的readTimeout 时间

异常:org.springframework.dao.DataAccessResourceFailureException: Connection is closed; nested exception is java.lang.RuntimeException: Connection is closed

原因： 无法连接远程服务器

解决:

把ip地址重新改好即可

异常:java.lang.IllegalStateException: No instances available for www.baidu.com

原因:使用了Ribbon进行负载均衡。@LoadBalanced负载均衡 必须使用注册中心上的服务名 不能用ip或域名

解决先删除@LoadBalanced注解即可

c.a.c.n.c.NacosPropertySourceBuilder : parse data from Nacos error,dataId:xiaohua-service-dev.yaml,data:spring:

原因: 配置内容编码不一致导致。

配置文件中有中文，而nacos读取配置文件时默认编码为utf-8，而通过cmd启动项目默认使用gbk。

解决方案

把nacos 配置中心的注解 修改为英文,或删除

异常:java.lang.IllegalStateException: No Feign Client for loadBalancing defined. Did you forget to include spring-cloud-starter-loadbalancer?

原因: 由于SpringCloud Feign在Hoxton.M2 RELEASED版本之后不再使用Ribbon而是使用spring-cloud-loadbalancer，所以不引入spring-cloud-loadbalancer会报错，于是看到希望的我加了一下如下依赖

解决方案: Hoxton.SR8 版本的spring-cloud-dependencies 添加如上依赖 即可解决

BCrypt.hashpw 失败。
原因是 使用错的方法 获取 29位的盐,查找获取盐的方法.修改使用 gensalt() 即可。

异常 ‘url’ attribute is not specified and no embedded datasource could be configured.Reason: Failed to determine a suitable driver class

原因:这个项目中是分布式微服务开发，这几个模块共同组成了一个项目，服务A依赖服务B,或者其他模块,这样就相当于一个项目中产生了2个application.yml文件，微服务A的application.yml文件把 微服务B的application.yml配置给覆盖掉了，所以产生了以上异常。

异常: java.io.IOException,freemarker.template.TemplateException:Template not found for name “article.ftl”.The name was interpreted by this TemplateLoader: MultiTemplateLoader(loader1 = org.springframework.ui.freemarker.SpringTemplateLoader@3a988299, loader2 = ClassTemplateLoader(resourceLoaderClass=org.springframework.web.servlet.view.freemarker.FreeMarkerConfigurer, basePackagePath=”” /* relatively to resourceLoaderClass pkg */)).

原因分析: 没有找到article.ftl 模板 导致 IO异常。。

解决:
解决: 检查template-loader-path: classpath:/article/ 属性, !!!! 要在classpath:/xxx/ 最后加 “/“

后备方案:在classpath下创建templates文件夹, 再ftl模板拉到文件夹内。

异常:io.minio.errors.InvalidResponseException: Non-XML response from server

原因

解决 修改minio 容器的 映射端口(!!! API 端口 和 Console 端口都要 映射)
