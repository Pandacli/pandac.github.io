01.Spring Data 简介

what：是什么

Spring Data是一个用于简化持久层数据访问的开源框架。
其主要目标是使得对数据的访问变得方便快捷。
Spring Data可以极大的简化数据操作的写法，可以在几乎不用写实现的情况下，实现对数据的访问和操作。
包括CRUD外，还包括如分页、排序等一些常用的功能，几乎可以节省持久层代码80%以上的编码工作量。
Spring Data的官网：http://projects.spring.io/spring-data/

02.Spring Data ElasticSearch简介

what：是什么

  • Templates as a high-level abstraction for storing, searching, sorting documents and building aggregations.
    模板类 ElasticsearchRestTemplate 作为存储、搜索、排序文档和构建聚合的高级抽象
  • Repositories which for example enable the user to express queries by defining interfaces having customized method names
    允许用户通过定义具有自定义方法名的接口来表示查询的存储库
  • Spring Data ElasticSearch 基于 spring data API 简化 elasticSearch操作，将原始操作elasticSearch的客户端API 进行封装 。
  • Spring Data为Elasticsearch项目提供集成搜索引擎。
  • Spring Data Elasticsearch POJO的关键功能区域为中心的模型与Elastichsearch交互文档和轻松地编写一个存储库数据访问层。

官方网站：http://projects.spring.io/spring-data-elasticsearch/

why：为什么使用

用来操作ElasticSearch的框架，使得开发更加便捷

05 Elasticsearch 对象映射

编写ES实体类 与索引库的_resources字段建立映射。

简单介绍常用的数据映射

普通数据映射
@Document：声明索引库配置
1.indexName：索引库名称
2.type：类型名称，默认是“docs”
3.shards：分片数量，默认5
4.replicas：副本数量，默认1
@Id：声明实体类的id
@Transient: 默认情况下,所有字段在存储检索时都映射到文档,但排除本注释的字段
@Field：声明字段属性
1.type：字段 如果未指定字段类型，则默认为FieldType.Auto
2.analyzer：指定分词器类型
3.index：是否创建索引 默认为true
4.store:是否存储 默认为false
@GeoPointField : 该字段为 经纬度数据类型。如果字段是类的实例，则可以省略GeoPoint。

范围映射
当字段使用Integer_Range、Float_Range、Long_Range、Double_Range、Date_Range或Ip_Range中的一种类型进行注释时，该字段必须是将映射到 Elasticsearch 范围的类的实例

7.ElasticSearch Operations (core模块)

IndexOperations定义索引级别的操作，例如创建或删除索引。

DocumentOperations 提供了 新增、更新、删除、getById es文档的操作。

SearchOperations提供了查询搜索es文档的操作

ElasticsearchOperations API结合DocumentOperations和SearchOperations接口。

7.1 core.query 搜索

SearchOperations和ReactiveSearchOperations接口中定义的几乎所有方法都带有一个Query参数，该参数定义要执行的查询以进行搜索。Query是一个接口，Spring Data Elasticsearch 提供了三种实现CriteriaQuery：StringQuery和NativeSearchQuery

  • CriteriaQuery:条件查询,实例化Criteria使用其方法来进行查询。
    Criteria API
  • StringQuery: 字符串查询
  • NativeSearchQuery:组合查询API
    使用原生的的QueryBuidlers生成不同的Query, 传入NativeSearchQuery 进行不同的查询。

or @Query annotated methods
